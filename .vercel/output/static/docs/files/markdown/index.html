Markdown

在您的 Nuxt 应用程序中创建和查询 Markdown 文件，并使用 MDC 语法集成 Vue 组件。

使用方法

定义集合

content.config.ts
import { defineCollection, defineContentConfig } from '@nuxt/content'
import { z } from 'zod'

export default defineContentConfig({
  collections: {
    blog: defineCollection({
      type: 'page',
      source: 'blog/*.md',
      schema: z.object({
        date: z.string()
      })
    })
  }
})
了解更多关于 page 集合类型的信息。

创建 .md 文件

在 content/blog/ 目录中创建博客文章。

---
date: 2020-11-11
---

# Foo

这是 Foo 博客文章。

---
date: 2024-12-12
---
你好
我是 bar。很高兴认识你。

查询 Markdown 文件

现在我们可以查询博客文章：

// 获取 foo 文章
const fooPost = await queryCollection('blog').path('/blog/foo').first()

// 查找所有文章
const allPosts = await queryCollection('blog').order('date', 'DESC').all()

显示 Markdown

要显示 markdown 文件的内容，您可以使用 <ContentRenderer> 组件。

blog/[slug].vue
<script setup>
const slug = useRoute().params.slug
const { data: post } = await useAsyncData(`blog-${slug}`, () => {
  return queryCollection('blog').path(`/blog/${slug}`).first()
})
</script>

<template>
  <!-- 将博客文章渲染为 Prose 和 Vue 组件 -->
  <ContentRenderer :value="post" />
</template>
了解更多关于 <ContentRenderer> 组件和 Prose 组件 的信息。

前置内容

前置内容是基于 Markdown 的 CMS 提供页面元数据的约定，如描述或标题。在 Nuxt Content 中，前置内容使用带有 key: value 对的 YAML 语法。

这些数据在渲染内容时可用，并且可以存储您需要的任何信息。

语法

您可以使用 --- 标识符在 content/ 目录中的 Markdown 文件顶部声明前置内容块。

content/index.md
---
title: '页面标题'
description: '页面的元描述'
---

<!-- 页面内容 -->
example.ts
const home = await queryCollection('content').path('/').first()

console.log(home.title)
// => '页面标题'
console.log(home.description)
// => '页面的元描述'
console.log(home.body)
// => 页面内容的 AST 对象

原生参数

键类型默认值描述
titlestring页面的第一个 <h1>页面标题，也会注入到元数据中
descriptionstring页面的第一个 <p>页面描述，将显示在标题下方并注入到元数据中
navigationbooleantrue定义页面是否包含在 queryCollectionNavigation 返回值中。
您在前置内容块中定义的额外参数需要在您的模式中定义（参见本页顶部示例中的 date 参数），才能用于查询。

MDC 语法

我们创建了 MDC 语法来增强 Markdown 功能，使您能够在 Markdown 中集成带有插槽和属性的 Vue 组件。

探索完整的 MDC 文档。

安装 MDC VS Code 扩展 以获得 MDC 语法的适当语法高亮。

Vue 组件

您可以在 Markdown 文件中使用任何 Vue 组件。

我们有特殊的语法，使您在 Markdown 文件中使用组件更加容易。

content/index.md
::component-name
默认插槽内容
::
如果您不使用 components/content/ 目录，则在 Markdown 中使用的组件必须在您的 Nuxt 应用程序中标记为 global。访问 Nuxt 3 文档 了解更多信息。

块组件

块组件是接受 Markdown 内容或另一个组件作为插槽的组件。

组件必须包含至少一个 <slot /> 组件来接受格式化文本。

在 markdown 文件中，使用带有 :: 标识符的组件。

::card
卡片内容
::

<!-- components/content/Card.vue -->
<template>
  <div class="p-2 border bg-white dark:bg-black dark:border-gray-700 rounded">
    <slot />
  </div>
</template>

卡片内容

插槽

组件的插槽可以接受内容或其他组件。

  • 默认插槽 渲染块组件内的顶级内容或使用 #default
  • 命名插槽 使用 # 标识符渲染相应的内容。
::hero
我的页面标题

#description
这将在 `description` 插槽内渲染。
::

<template>
  <section>
    <h1 class="text-4xl">
      <slot mdc-unwrap="p" />
    </h1>
    <slot name="description" />
  </section>
</template>

我的页面标题

了解更多关于 <slot /> 组件的信息。
您可以在组件插槽内使用 Markdown：
::my-title
[富文本](/) 将被组件**渲染**。
::

<template>
  <h1 class="text-4xl">
    <slot mdc-unwrap="p" />
  </h1>
</template>

属性

有两种方法可以使用 MDC 将属性传递给组件。

内联方法

{} 标识符通过使用 key=value 语法以简洁的方式将属性传递给组件。

::alert{type="warning"}
**alert** 组件。
::

<script setup>
const props = defineProps({ type: { type: String } })

const alertClass = computed(() => {
  return {
    warning: 'bg-orange-100 border-orange-200 dark:bg-orange-900 dark:border-orange-800',
    info: 'bg-blue-100 border-blue-200 dark:bg-blue-900 dark:border-blue-800',
    success: 'bg-green-100 border-green-200 dark:bg-green-900 dark:border-green-800',
  }[props.type]
})
</script>

<template>
  <div
    class="text-black p-2 border dark:text-white rounded"
    :class="alertClass"
  >
    <slot mdc-unwrap="p" />
  </div>
</template>

alert 组件。

多个属性可以用空格分隔：

::alert{type="warning" icon="exclamation-circle"}
糟糕！发生了错误
::

v-bind 简写 : 也可以用于将属性绑定到前置内容中的值。

---
type: "warning"
---

::alert{:type="type"}
您的警告
::

如果您想将数组或对象作为属性传递给组件，您可以将它们作为 JSON 字符串传递，并在属性键前加上冒号以自动解码 JSON 字符串。请注意，在这种情况下，您应该为值字符串使用单引号，这样您就可以使用双引号来传递有效的 JSON 字符串：

::dropdown{:items='["Nuxt", "Vue", "React"]'}
::

::dropdown{:items='[1,2,3.5]'}
::

::chart{:options='{"responsive": true, "scales": {"y": {"beginAtZero": true}}}'}
::
YAML 方法

YAML 方法使用 --- 标识符每行声明一个属性，这对于可读性很有用。

::icon-card
---
icon: IconNuxt
description: 利用 Nuxt 和 Nuxt 生态系统的全部力量。
title: Nuxt 架构。
---
::

<script setup>
defineProps({
  title: {
    type: String,
    default: '默认标题'
  },
  description: {
    type: String,
    default: '默认描述'
  },
  icon: {
    type: String,
    default: 'IconMarkdown'
  }
})
</script>

<template>
  <div class="p-6 border bg-white dark:bg-black dark:border-gray-700 rounded">
    <component :is="icon" class="w-20 h-20" />
    <h2 class="text-3xl font-semibold mb-2">
      {{ title }}
    </h2>
    <p>{{ description }}</p>
  </div>
</template>

属性

属性对于突出显示和修改段落部分很有用。语法几乎与内联组件和 markdown 链接语法相似。

可能的值是所有命名属性、使用 .class-name 表示法的类和使用 #id-name 的 ID。

你好 [世界]{style="color: green;" .custom-class #custom-id}!

你好 世界 !

除了 mdc 组件和 span，属性语法将适用于图像、链接、内联 code、粗体 和 斜体 文本。

属性适用于：

- [属性](#attributes){style="background-color: green;"}, `code`{style="color: cyan;"},
- _斜体_{style="background-color: yellow; color:black;"} 和 **粗体**{style="background-color: lightgreen;"} 文本。

属性适用于：

  • 属性, code,
  • 斜体 和 粗体 文本。

数据绑定

您可以使用 {{ $doc.variable || 'defaultValue' }} 语法在 Markdown 文档中绑定数据。这些值可以在文档顶部的 YAML 前置内容中定义，在每个 MDC 组件内定义，或者使用 <ContentRenderer> 组件的 data 属性注入。

在 YAML 中定义

---
title: '页面标题'
description: '页面的元描述'
customVariable: '自定义值'
---

# 标题是 {{ $doc.title }}，customVariable 是 {{ $doc.customVariable || 'defaultValue' }}

使用 <ContentRenderer> 在外部定义

test.vue
<template>
  <div>
    <ContentRenderer :value="data" :data="mdcVars"/>
    <button type="button" v-on:click="mdcVars.name = 'Hugo'">更改名称</button>
  </div>
</template>

<script setup lang="ts">
const { data } = await useAsyncData(() => queryCollection('content').path('/test').first());
const mdcVars = ref({ name: 'Maxime'});
</script>
test.md
# 你好 {{ $doc.name || '世界' }}

Prose 组件

在 Nuxt Content 中，prose 表示由 Markdown 语法生成的 HTML 标签，如标题级别和链接。

对于每个 HTML 标签，使用一个 Vue 组件，允许您在需要时覆盖它们，例如 <p> 变为 <ProseP>。

如果您想自定义 Prose 组件，以下是推荐的步骤：

  • 查看原始组件源代码。
  • 使用完全相同的属性。
  • 在您的 components/content/ 目录中，给它相同的名称。
  • 使它成为您的 🚀。
在 Prose 组件部分阅读完整的 Prose 参考。

代码高亮

Nuxt Content 使用 Shiki，它使用 VSCode 主题为标记着色。

代码高亮适用于 ProsePre 和 ProseCode。

代码块的每一行在 line 属性中获取其行号，因此可以标记或单独设置行的样式。

阅读 API 参考以配置或完全禁用语法高亮。

图像

您可以将图像添加到您的 public 目录：

目录结构
content/
  index.md
public/
  image.png
nuxt.config.ts
package.json

然后在 content 目录中的 markdown 文件中这样使用它们：

content/index.md
![我的图像](/image.png)

摘录

内容摘录或摘要可以使用 <!--more--> 作为分隔符从内容中提取。

content/index.md
---
title: 介绍
---

学习如何使用 `@nuxt/content`。

<!--more-->

超过更多分隔符的完整内容。

描述属性将包含摘录内容，除非在前置内容属性中定义。

如果文本中没有 <!--more--> 分隔符，则摘录未定义。

如果您想使用摘录功能，应该在集合模式中定义 excerpt 字段。
content.config.ts
const content = defineCollection({
  type: 'page',
  source: '**',
  schema: z.object({
    excerpt: z.object({
      type: z.string(),
      children: z.any(),
    }),
  }),
})
了解更多关于集合模式的信息。

示例变量将被注入到文档中：

{
  "excerpt": Object
  "body": Object
  // ... 其他键
}
