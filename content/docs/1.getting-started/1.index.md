欢迎使用 Nuxt Content v3，这是一个重大升级，为您的 Nuxt 项目带来了增强的性能和创新功能。我们的 Git 基础 CMS 的最新版本针对现代应用开发进行了优化。

## 新特性

### 内容集合

集合可以组织项目中的相关项目，帮助您更高效地管理大型数据集。主要优势包括：

- **结构化数据**：在 [`content.config.ts`](/docs/collections/define#defining-collections) 中配置数据库架构和定义集合
- **类型安全查询**：所有工具都直接集成 TypeScript
- **自动验证**：确保前置元数据字段和数据文件（json、yml...）的数据一致性
- **高级查询构建器**：轻松筛选、排序和分页您的集合
- **Studio 集成**：通过 [Studio](/studio) 增强表单生成和优化编辑体验

了解有关 [内容集合](/docs/collections/define) 的更多信息。

### 提升的性能

v2 中的一个重大挑战是存储文件所需的大型打包大小，特别影响无服务器部署。

V3 通过在生产环境中过渡到基于 SQL 的存储来解决这个问题。此切换不需要任何配置，支持开发模式、静态生成、服务器托管、无服务器和边缘部署。

::prose-note
新的数据库系统增强了您的数据文件的存储和结构方式，确保更好的性能和可扩展性。此更新完全在后台进行，不会影响您可以在 Content 中使用的文件类型（`yml`、`json` 和 `markdown`）。
::

优势包括：

- **优化查询**：SQL 存储实现超快数据检索
- **通用兼容性**：我们的基于适配器的系统在所有部署模式（[服务器](/docs/deploy/server)、[无服务器](/docs/deploy/serverless) 和 [静态](/docs/deploy/static)）中集成 SQL 数据库。我们欢迎社区贡献更多适配器。

### TypeScript 集成

新的集合系统为所有数据提供自动 TypeScript 类型。每个工具和 API 都基于您的集合定义进行强类型化，确保整个开发过程中的类型安全。

### Nuxt Studio 集成 :badge[即将推出]{color="neutral"}

[Nuxt Studio](/docs/studio/setup) 和 v3 设计为完美互补。[studio 模块](https://github.com/nuxt-content/studio) 正在创建一个理想环境，让开发人员可以专注于代码，而团队成员可以通过直观的界面管理内容。
