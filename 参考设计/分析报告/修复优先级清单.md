## 🔴 严重问题（Critical）

### 1. `utils/pluginData.ts` — 重复的应用 ID + 无效分类

**文件**：`utils/pluginData.ts`

`apps` 数组中 `id: 5` 出现了**两次**，分别对应两个不同的应用：

```ts
// 第一处 id: 5
{
  id: 5,
  name: '万象漫剧短剧创作',
  category: '漫剧创作',   // ⚠️ 此分类不存在于 categories 数组中
  ...
},
// 第二处 id: 5 —— 重复！
{
  id: 5,
  name: 'AI证件照',
  ...
},
```

**影响**：
- Vue 在使用 `v-for :key="app.id"` 时会产生运行时警告，并可能引发渲染错乱。
- `plugin.vue` 的筛选逻辑依赖 `app.id <= 3`（官方推荐）和 `app.id >= 50`（独立应用），重复 ID 会导致逻辑错误。
- `category: '漫剧创作'` 不在 `categories` 定义中，该应用在任何分类筛选下都无法出现（除"全部应用"外）。

**修复方案**：
1. 将第二个 `id: 5` 改为唯一 ID（如 `id: 6`），并将后续所有 ID 依次递增。
2. 将 `category: '漫剧创作'` 改为已定义的有效分类（如 `'video'`），或在 `categories` 数组中补充此分类。

### 7. AOS 在 `buidai.vue` 中重复初始化，且参数不一致

**文件**：`app.vue`、`pages/buidai.vue`

`app.vue` 已在全局 `onMounted` 中通过 `utils/aos.ts` 的 `initAOS()` 初始化 AOS（`duration: 800`），`buidai.vue` 又额外再次导入并以不同参数手动初始化：

```ts
// app.vue —— 全局初始化
onMounted(() => {
  initAOS() // duration: 800, easing: 'ease-out-cubic'
})

// pages/buidai.vue —— 重复初始化，参数不同！
import AOS from 'aos'
import 'aos/dist/aos.css'
// ...
AOS.init({ duration: 500, once: true, offset: 50 })
```

**影响**：后一次初始化会覆盖全局配置，导致站内各页面动画行为不一致，且 `buidai.vue` 额外引入了 AOS CSS，造成重复加载。

**修复方案**：删除 `buidai.vue` 中对 AOS 的导入和 `AOS.init()` 调用，依赖全局的 `initAOS()`。

### 8. `process.dev` 已弃用（Nuxt 4 规范）

**文件**：`error.vue`

```ts
const isDev = process.dev  // ⚠️ Nuxt 4 中已废弃
```

Nuxt 4 应使用 `import.meta.dev`，`process.dev` 是向后兼容 API，可能在未来的 Nuxt 版本中移除。

**修复方案**：

```ts
const isDev = import.meta.dev
```

### 9. `window.pageYOffset` 已废弃（Web 标准）

**文件**：`components/BackToTop.vue`

```ts
isVisible.value = window.pageYOffset > 300  // ⚠️ 已从 Web 标准中废弃
```

`window.pageYOffset` 是 `window.scrollY` 的别名，但已在 Web 标准中被标记为废弃属性。

**修复方案**：

```ts
isVisible.value = window.scrollY > 300
```

### 10. Tailwind 配置版本不一致

**文件**：`package.json`、`tailwind.config.js`

`package.json` 中安装的是 **Tailwind CSS v4**（`"tailwindcss": "^4.1.18"`），但 `tailwind.config.js` 使用的是 **v3 的 JS 配置语法**。Tailwind v4 已迁移为 CSS-first 配置，`tailwind.config.js` 中自定义的颜色（`primary`、`accent`、`secondary`）和动画（`marquee-vertical` 等）可能未正确注入到最终产物中，需要验证是否实际生效。

**修复方案**：将 `tailwind.config.js` 中的自定义配置迁移到 `assets/css/main.css` 中，使用 CSS 变量和 `@theme` 指令按照 Tailwind v4 规范重新定义。






### 14. 图片文件名包含中文字符

**文件**：`public/` 目录

多个公共资源文件使用了中文命名，例如：
- `public/images/大语言模型.jpeg`
- `public/images/企业级组织.png`
- `public/images/多模态交互支持.png`
- `public/plugin/万象漫剧.png`
- `public/plugin/即梦AI绘画.png`
- ... 等

在部分 Linux 服务器、CDN 或 Windows/macOS 混合开发环境中，中文路径可能引发文件系统编码问题，URL 中的中文也需要 percent-encoding，影响稳定性。

**修复方案**：将中文文件名统一重命名为英文或拼音，并更新代码中所有引用路径。

### 15. `solutions.vue` 完全缺少暗色模式支持

**文件**：`pages/solutions.vue`

整个页面没有任何 `dark:` 前缀的 Tailwind 类，而其他大多数页面（`plugin.vue`、`pricing.vue` 等）均已适配暗色模式。用户开启暗色模式时，`solutions.vue` 将以纯白背景强制显示，视觉体验割裂。

**修复方案**：为页面关键容器和文字元素补充 `dark:` 对应样式类。
